# カード効果作成 — 依頼・設計ガイド

身内語録大戦（仮）にカードを追加・依頼するときに必要な情報をまとめたドキュメントです。  
**実装者向けの JSON 構文**は [CARD_FORMAT.md](./CARD_FORMAT.md)、**実データ**は [cards.json](./cards.json) を参照してください。

---

## このゲームの前提（効果設計の土台）

| 項目 | 内容 |
|---|---|
| 勝利条件 | 相手ライフを 0 にする |
| ライフ | 初期 20 |
| リソース | **空気**（コスト）。先攻初手 3 → 毎ターン上限 +1（最大 10）／後攻初手 4 |
| デッキ | 40 枚・同一カード最大 4 枚 |
| 手札 | 開始時 5 枚ドロー、以降ターン開始ごとに 1 枚 |
| 山札切れ | ターン開始ドロー時に引けなければ敗北（捨て札は山札に戻さない） |
| ターン | 交互。自分のターン中、空気が続く限りカードを何枚でも使用可 |

カードはイラストなし。**語録テキスト**と**発言者**が主役です。

---

## 依頼時に必ず書いてほしいこと

新規カードを誰かに作ってもらう／自分で設計するとき、次の項目を埋めて渡してください。

### 必須

| # | 項目 | 説明・例 |
|---|---|---|
| 1 | **語録（text）** | 画面上に大きく出るセリフ。「またそれ言うの！？」 |
| 2 | **発言者（speaker）** | 誰の語録か。「A君」「部長」 |
| 3 | **コスト（cost）** | 0〜10 程度の整数。高いほど強い想定 |
| 4 | **属性（tone）** | 下表から **1 つ**（熱量 / 冷徹 / 泥沼 / 口癖） |
| 5 | **メイン効果（effect）** | プレイ時に起きること（日本語で OK） |
| 6 | **ID（id）** | 英小文字＋アンダースコア。例: `a_mata_ka` |

### 任意だがあると実装が速い

| 項目 | 説明 |
|---|---|
| **発言者コンボ（speaker_effect）** | 直前と同じ発言者のカードの直後に使ったときの追加効果 |
| **effect_note / combo_note** | 人間向けの効果メモ（画面下やリスト用） |
| **想定レアリティ・役割** | 序盤用 / フィニッシャー / 妨害 など |
| **同発言者とのシナジー** | どの speaker と組ませたいか |

### 依頼テンプレート（コピー用）

```
【カード依頼】

■ 語録:
■ 発言者:
■ コスト:
■ 属性（熱量/冷徹/泥沼/口癖）:
■ ID案:

■ メイン効果（プレイ時）:
  例）相手に2ダメージ
  例）自身を3回復し、相手にミュート

■ 発言者コンボ（任意）:
  例）同じ発言者の直後ならコスト-1（標準）
  例）同じ発言者の直後ならドロー1
  例）同じ発言者の直後ならダメージ2倍

■  flavor・背景（任意）:
■  バランス上の注意（任意）:
```

## 発言トーン（属性）— 1 枚につき 1 つ

設計の「色」です。日和デバフは **熱量** カードだけを封じます。

| キー | 名称 | 語録のイメージ | 効果の方向性 |
|---|---|---|---|
| `passion` | **熱量** | 勢い・怒り・爆笑・声が大きい | 直接ダメージ、コストを踏み倒すラッシュ |
| `logical` | **冷徹** | マジレス・正論・呆れ・冷めた反応 | 妨害、手札操作、デバフ付与 |
| `chaos` | **泥沼** | 言い訳・意味不明・深夜ノリ・嘘 | 回復、トリッキーな効果、リスク付き大技 |
| `habit` | **口癖** | 繰り返しの決まり文句 | **同じ発言者の連鎖（コンボ）と相性が良い** |

## 発言者コンボ（speaker_effect）

**自分のターン中**、直前にプレイしたカードと **同じ speaker** のカードを続けて使うと発動します。

| ルール | 内容 |
|---|---|
| デフォルト | コスト **-1**（最低 0） |
| 上書き | カードごとに `cost_reduction` / `damage_multiplier` / 追加 `effects` を指定可能 |
| ミュート中 | コンボは一切発動しない |
| ターン開始 | 直前発言者はリセットされる |

依頼文では「標準コンボ（コスト-1）」か、特別なコンボかを明示してください。

## 状態異常（デバフ）

プレイヤーに付与する効果。カード文言では **ツボ・日和・ミュート** と表示されます。

| キー | 名称 | 効果 |
|---|---|---|
| `tsubo` | **ツボ**（爆笑） | 付与中、**カードを使うたびに** 自身ライフ 1 ダメージ |
| `hiyori` | **日和** | 付与中、**熱量**属性のカードが使えない |
| `mute` | **ミュート** | **次の自分ターン**、発言者コンボが発動しない |

付与時は「何ターン」を指定（ミュートは常に次の 1 ターン）。

依頼例:
- 「相手にツボ 2 ターン」
- 「相手に日和 2 ターン」
- 「相手にミュート」

## 実装できる効果一覧

依頼は日本語で書き、実装時に下の型に落とし込みます。

### 基本効果

| 依頼の書き方 | 意味 |
|---|---|
| 相手に N ダメージ | 相手ライフを N 減らす |
| 自身を N 回復 | 自身ライフを N 回復（上限 20） |
| カードを N 枚ドロー | 山札から引く（山札が空なら引けない） |
| 自身の手札をランダムに N 枚捨てる | `discardSelf` |
| 自身に N ダメージ | 自傷（「条件」ではない通常ダメージ） |

### 条件付き（○○のときだけ）

「条件」という曖昧な言い方ではなく、**具体的な条件**を書いてください。

| 依頼の書き方 | 実装キー |
|---|---|
| 相手 HP が N 以下のとき、M ダメージ | `damageIf` + `opponentHpLte` |
| 相手 HP が N 以上のとき… | `opponentHpGte` |
| 自身 HP が N 以下のとき、M 回復 | `healIf` + `selfHpLte` |
| 相手の手札が N 枚以上のとき… | `opponentHandGte` |
| 相手の手札が N 枚以下のとき… | `opponentHandLte` |
| 自身の手札が N 枚以上のとき… | `selfHandGte` |
| 直前に自分が「○○」のカードを使っていたとき… | `selfLastSpeakerIs` |
| 相手の直前が「○○」のとき… | `opponentLastSpeakerIs` |

表示例: `相手HP10以下→6ダメ`（ゲーム内では「条件」という文字は使いません）

### 複数効果の組み合わせ

1 枚のカードに **効果を複数** 書けます（上から順に処理）。

依頼例（`b_finisher`）:
> 相手 HP 10 以下なら 6 ダメージ。さらに常に相手に 3 ダメージ。

## バランス・制約の目安

| 目安 | 内容 |
|---|---|
| コスト 0〜1 | ダメージ 1〜2、小さいユーティリティ |
| コスト 2〜3 | ダメージ 2〜4、またはドロー＋小ダメ |
| コスト 4〜5 | フィニッシャー級（条件付き大ダメなど） |
| コンボ依存 | 単体弱め＋コンボで強くする設計が可能 |
| 完全上位互換 | 同コストで既存カードを全面に上回るものは不可 |

既存カード一覧・数値感は [cards.json](./cards.json) を参照してください。

## 実装フロー（依頼 → ゲーム反映）

1. 上記テンプレートで設計を渡す  
2. 実装者が [cards.json](./cards.json) に 1 件追加  
3. `implemented: true`, `excluded: false` で有効化  
4. プレイテスト後 `bug_free: true` に更新  
5. デッキに入れる場合は [initial-deck.json](./initial-deck.json) も更新（40 枚・同名 4 枚まで）

## 依頼時の NG 例

- 「強いカードを作って」だけ（数値・役割が不明）
- 属性を 2 つ以上付けたい（システム上 **1 枚 1 属性**）
- 永続バフ・フィールドカードなど未実装の仕組み（要相談）
- 交戦力・同時行動など **旧ルール** の用語（現在はターン制・直接ダメージ）

未実装の効果が必要な場合は、依頼文に **「新規効果が必要」** と明記し、望む挙動を文章で書いてください。
